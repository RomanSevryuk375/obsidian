### Суть (The "What")
**Структура** — это пользовательский значимый тип ([[Value Types|value type]]). В отличие от [[Классы и объекты|классов]], переменные структур содержат свои данные непосредственно ("in-place"), а не ссылку на них. 

Структуры решают задачу **локальности данных** и **снижения нагрузки на Garbage Collector ([[GC]])**. Они идеально подходят для небольших объектов, которые ведут себя как простые значения (числа, точки на плоскости, цвета), где стоимость копирования значения меньше стоимости управления ссылкой в куче.

---

### Как это работает (The "How" / Nutshell style)

#### 1. Управление памятью и [[CLR]]
*   **Размещение:** Структуры могут располагаться как в стеке (Stack), так и в куче (Heap). Если структура — локальная переменная, она живет в стеке. Если она является полем класса, она встраивается непосредственно в память этого класса в куче.
*   **Копирование:** При присваивании (`s1 = s2`) или передаче в метод происходит **побитовое копирование** всей структуры. Это "поверхностное" копирование: если внутри структуры есть ссылка на объект, скопируется только адрес.
*   **Отсутствие заголовка:** У структур нет накладных расходов в 16 байт (Sync Block Index и Type Handle), которые есть у каждого объекта в куче.

#### 2. Особенности наследования
Все структуры неявно наследуются от `System.ValueType`, который переопределяет методы `Equals` и `GetHashCode` для сравнения по значениям полей (используя рефлексию, если не переопределено вручную). 
*   Структуры являются **запечатанными (sealed)**: от них нельзя наследоваться, и они не могут наследоваться от других классов/структур (только реализовывать интерфейсы).

#### 3. Специальные виды структур
*   **readonly struct:** Гарантирует неизменяемость всех полей. Позволяет компилятору избегать создания "защитных копий" (defensive copies) при доступе, что критично для производительности.
*   **ref struct:** Тип, который гарантированно живет только в стеке (например, `Span<T>`). Не может быть упакован ([[boxing]]) или быть полем обычного класса.

> [!INFO] Нюанс: Конструктор по умолчанию
> До C# 10 структуры не могли иметь явный конструктор без параметров. Теперь это разрешено, но важно помнить: оператор `default(MyStruct)` всё равно создаст объект, занулив все поля в обход вашего конструктора.

### Пример кода (C# 12)

```csharp
using System;

// Применение Primary Constructor (C# 12) в структуре
// Пометка readonly — рекомендация для большинства структур
public readonly struct Vector2D(double x, double y) : IEquatable<Vector2D>
{
    public double X { get; init; } = x;
    public double Y { get; init; } = y;

    // Вычисляемое свойство
    public double Magnitude => Math.Sqrt(X * X + Y * Y);

    // Переопределение Equals для исключения рефлексии (Nutshell style)
    public bool Equals(Vector2D other) => X == other.X && Y == other.Y;

    public override bool Equals(object? obj) => obj is Vector2D other && Equals(other);

    public override int GetHashCode() => HashCode.Combine(X, Y);

    public static Vector2D operator +(Vector2D a, Vector2D b) => 
        new(a.X + b.X, a.Y + b.Y);
}

public class Program
{
    public static void Main()
    {
        Vector2D v1 = new(3.0, 4.0);
        Vector2D v2 = v1; // Полное копирование данных в стеке
        
        var v3 = v1 + v2;
        
        Console.WriteLine($"V3: {v3.X}, {v3.Y}, Mag: {v3.Magnitude}");
    }
}
```

### Ошибки и Best Practices

> [!DANGER] Anti-pattern: Mutable Structs (Изменяемые структуры)
> Избегайте создания структур с изменяемыми полями (публичными сеттерами). Это ведет к непредсказуемым багам, когда вы меняете копию структуры, думая, что меняете оригинал (например, при работе со свойствами или элементами списка). Всегда делайте структуры `readonly`.

> [!WARNING] Ошибка: Упаковка (Boxing)
> При передаче структуры в метод, принимающий `object` или интерфейс, происходит упаковка (выделение памяти в куче). Если структура часто упаковывается, преимущество в производительности теряется. Используйте обобщения (`Generic<T> where T : struct`).

**Best Practices:**
1.  **Размер имеет значение:** Используйте структуры только если размер данных не превышает **16-24 байта** (примерно 2-3 указателя). Передача крупных структур по значению дороже, чем работа со ссылками.
2.  **Передача по ссылке:** Если структура велика или вы хотите избежать копирования, используйте модификаторы `in` (для чтения) или `ref` (для изменения).
3.  **Default State:** Всегда проектируйте структуру так, чтобы состояние по умолчанию (все поля = 0/null) было валидным.
4.  **Value Equality:** Всегда переопределяйте `Equals` и `GetHashCode`, чтобы избежать медленной стандартной реализации на основе рефлексии.

> [!TIP] Нюанс: Массивы структур
> Массив структур (например, `Vector2D[]`) хранится в куче как один непрерывный блок памяти. Это крайне эффективно для кэша процессора (CPU Cache Locality), в отличие от массива классов, который является массивом ссылок на разбросанные объекты.
