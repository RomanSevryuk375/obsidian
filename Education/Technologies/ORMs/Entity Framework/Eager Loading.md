**Eager Loading (Жадная загрузка)** — это техника загрузки связанных данных **вместе** с основной сущностью в одном запросе из [[БД]].2026-02-28 10:31

В [[Entity Framework|EF Core]] это делается с помощью методов `Include` и `ThenInclude`.

Чтобы понять суть, давай сначала посмотрим на проблему, которую это решает.

---

### 1. Синтаксис `Include` (Уровень 1)

Метод `Include` загружает данные, связанные напрямую с сущностью, которую ты запрашиваешь.

```csharp
var bills = context.Bills
    .Include(b => b.Status) // Загрузить статус (One-to-One / Many-to-One)
    .Include(b => b.Payments) // Загрузить платежи (One-to-Many)
    .ToList();
```

**Что происходит в SQL?**
[[Entity Framework|EF]] генерирует [[LEFT JOIN]].
```sql
SELECT b.*, s.*, p.*
FROM Bills b
LEFT JOIN BillStatuses s ON b.StatusId = s.Id
LEFT JOIN PaymentNotes p ON b.Id = p.BillId
```

---

### 2. Синтаксис `ThenInclude` (Уровень 2 и глубже)

А что, если тебе нужно загрузить данные "вглубь"?
Например: `Клиент` -> `Заказы` -> `Товары в заказе`.

Здесь `Include` уже не поможет, так как после `.Include(c => c.Orders)` мы "переключились" на контекст списка заказов. Нам нужен `ThenInclude`.

```csharp
var customers = context.Customers
    .Include(c => c.Orders)             // Уровень 1: Загружаем заказы
        .ThenInclude(o => o.OrderItems) // Уровень 2: Внутри заказов загружаем позиции
            .ThenInclude(oi => oi.Product) // Уровень 3: Внутри позиций загружаем сам Товар
    .ToList();
```

**Важное правило:**
*   `Include` всегда начинается от корня (от `Customer`).
*   `ThenInclude` продолжает цепочку от предыдущего шага.

Если тебе нужно загрузить и `OrderItems`, и `DeliveryInfo` для заказа (ветвление), ты делаешь так:

```csharp
var customers = context.Customers
    .Include(c => c.Orders)
        .ThenInclude(o => o.OrderItems) // Ветка 1
    .Include(c => c.Orders)             // Возвращаемся к корню, чтобы начать Ветку 2
        .ThenInclude(o => o.DeliveryInfo)
    .ToList();
```

---

### 3. Фильтрация внутри Include ([[Entity Framework|EF Core]] 5+)

Раньше `Include` грузил всё подряд. Теперь ты можешь фильтровать связанные данные прямо при загрузке.

*Пример: Загрузить Блоги, но вместе с ними только те Посты, у которых лайков > 100.*

```csharp
var blogs = context.Blogs
    .Include(b => b.Posts.Where(p => p.Likes > 100).OrderByDescending(p => p.Date))
    .ToList();
```
Это очень мощная фича, позволяющая не тянуть лишние мегабайты данных из базы.

---

### 4. Проблема: "Взрыв Декартова произведения" ([[Cartesian Explosion]])

`Include` — это не бесплатная магия.
Когда ты делаешь много `Include` для коллекций ([[One-to-Many]]), [[SQL]] возвращает огромную таблицу с дублирующимися данными.

Представь:
*   1 Пользователь
*   У него 10 Заказов
*   В каждом заказе 10 Товаров

Если сделать один запрос с джойнами, база вернет **1 * 10 * 10 = 100 строк**.
В каждой строке информация о Пользователе (Имя, Email...) будет дублироваться 100 раз. Это тратит трафик и память.

**Решение: [[AsSplitQuery()]]**
Начиная с [[Entity Framework|EF Core]] 5, ты можешь попросить разбить один гигантский запрос на несколько маленьких.

```csharp
var customers = context.Customers
    .Include(c => c.Orders)
    .ThenInclude(o => o.OrderItems)
    .AsSplitQuery() // <--- МАГИЯ ЗДЕСЬ
    .ToList();
```

**Что сделает EF Core:**
1.  Запрос 1: `SELECT * FROM Customers`
2.  Запрос 2: `SELECT * FROM Orders WHERE CustomerId IN (...)`
3.  Запрос 3: `SELECT * FROM OrderItems WHERE OrderId IN (...)`

В памяти EF Core сам соберет это в единое дерево объектов. Это часто работает **быстрее**, чем один огромный [[JOIN]].

### Резюме

1.  **`Include`** — для загрузки прямых связей (Счет -> Статус).
2.  **`ThenInclude`** — для погружения вглубь (Счет -> Платежи -> Автор платежа).
3.  **Filtered Include** — используй `.Where()` внутри Include, чтобы не грузить лишнее.
4.  **`AsSplitQuery`** — используй, если грузишь много больших списков (коллекций), чтобы избежать дублирования данных в [[SQ]]L ответе.

Применительно к твоему коду `Bills`:
```csharp
var bills = await _context.Bills
    .Include(b => b.Status)     // Чтобы видеть название статуса, а не только ID
    .Include(b => b.Order)      // Чтобы знать, за какой заказ счет
    .Include(b => b.Payments)   // Чтобы показать историю оплат
    .AsNoTracking()             // Для чтения всегда используй это (быстрее)
    .ToListAsync();
```
### 5. [[Explicit Loading]] vs Eager Loading vs [[Lazy Loading]]

| Тип          | Метод             | Кол-во запросов  | Когда использовать?                                                                     |
| :----------- | :---------------- | :--------------- | :-------------------------------------------------------------------------------------- |
| **Eager**    | `.Include()`      | 1 (обычно)       | Когда данные нужны **сразу и всегда** (например, вывести счет и его статус на экран).   |
| **Explicit** | `.Entry().Load()` | [[N+1]]          | Когда данные нужны **редко** или нужна **фильтрация** (только активные заказы клиента). |
| **Lazy**     | `virtual` св-ва   | [[N+1 ]](неявно) | В энтерпрайзе лучше **избегать**. Слишком легко случайно убить базу циклом foreach.     |
