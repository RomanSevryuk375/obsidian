**Entity Framework Core (EF Core)** — это современный, кроссплатформенный инструмент объектно-реляционного сопоставления ([[ORM]]) для .NET.

Простыми словами: это «переводчик» между твоим кодом на C# и базой данных. Вместо того чтобы писать сложные [[SQL]]-запросы (`SELECT * FROM Users...`), ты работаешь с обычными объектами C#, а EF Core сам генерирует и выполняет SQL за тебя.

---

### 1. Основная идея

Представь, что у тебя в коде есть класс `User`:
```csharp
public class User
{
    public int Id { get; set; }
    public string Name { get; set; }
}
```
И есть таблица `Users` в [[БД]]

Без [[ORM]] тебе нужно:
1. Открыть соединение с БД.
2. Написать [[SQL]]-запрос текстом.
3. Получить результат.
4. Вручную переложить данные из колонок таблицы в свойства объекта.

**С EF Core ты просто пишешь:**
```csharp
var users = context.Users.Where(u => u.Name == "Ivan").ToList();
```
EF Core сам превратит это в `SELECT * FROM Users WHERE Name = 'Ivan'`, выполнит запрос и вернет тебе список объектов.

### 2. Главные компоненты

*   **[[DbContext]] (Контекст базы данных):** Это главный класс, который управляет соединением с БД. Он действует как «мост» между твоим приложением и базой.
*   **[[DbSet]]<T>:** Представляет собой коллекцию всех сущностей определенного типа (аналог таблицы в [[БД]]).
*   **Entity (Сущность):** Класс C#, который сопоставляется со строкой в таблице.
