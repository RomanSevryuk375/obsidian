Команда трансформирует указанную директорию в Git-репозиторий, генерируя скрытую подпапку `.git`, которая содержит всю базу данных объектов и метаинформацию проекта.

---
### 1. Стандартная инициализация
Превращает текущую папку в рабочее дерево (Working Directory).

```bash
mkdir my-project
cd my-project

# Создает подпапку .git/
git init

# Вывод: Initialized empty Git repository in ...
```

### 2. С явным именем ветки (Best Practice)
Позволяет избежать legacy-имени `master` и сразу создать ветку `main` (или другую), игнорируя глобальные настройки [[Установка и настройка Git]].

```bash
# Инициализация с флагом --initial-branch
git init -b main
```

### 3. Bare репозиторий (DevOps паттерн)
Создает "голый" репозиторий **без рабочей директории** (без исходного кода, только БД Git). Используется на серверах в качестве центрального хранилища (remote origin) для приема [[git push]].

```bash
# Принято называть папки bare-репозиториев с суффиксом .git
git init --bare project-server.git
```

## Нюансы
*   **Безопасность повторного запуска:** Запуск `git init` в уже существующем репозитории безопасен. Он не перезапишет существующую историю или файлы, но может подтянуть новые шаблоны (templates).
*   **Структура:** Основная магия происходит внутри папки `.git`, где создаются `HEAD`, `objects/`, `refs/`. Удаление этой папки полностью стирает историю [[Git]], превращая проект в простой набор файлов.

## Связанные темы
*   [[git clone]] — альтернатива `init`: копирование существующего репозитория.
*   [[git status]] — проверка состояния репозитория сразу после создания.
*   [[git remote]] — добавление удаленного сервера к локальному репозиторию.
*   [[.gitignore]] — первый файл, который стоит создать после инициализации.
