# Что такое VCS и Git

**VCS** (Version Control System) — это система управления версиями, позволяющая отслеживать историю изменений и работать над кодом совместно, а **Git** — это распределенная (DVCS) реализация, где каждый разработчик хранит полную локальную копию всей истории проекта.

---

## Ключевые концепции

### Зачем нужен VCS
1.  **История и откат:** Замена файлов типа `main_final_v2.js` на строгую хронологию.
2.  **Командная работа:** [[Слияние изменений]] от разных людей без потери данных.
3.  **Ветвление:** Безопасные эксперименты в изолированных копиях кода.

### CVCS (SVN) vs DVCS (Git)
*   **Centralized (CVCS):** Один сервер хранит историю. Клиенты скачивают только последнюю версию. *Риск:* Если сервер падает, работа встает.
*   **Distributed (DVCS):** Сервер используется для синхронизации. Каждый клиент (`clone`) получает **полный бэкап** репозитория. *Плюс:* Работа офлайн, быстрые локальные операции.

### Архитектура Git (The Three States)
Git оперирует тремя зонами, через которые проходят файлы:
1.  **Working Directory:** Твоя рабочая папка с файлами (Sandbox).
2.  **Staging Area (Index):** Промежуточная зона, где формируется будущий коммит.
3.  **Repository (.git directory):** Место вечного хранения зафиксированных снимков (Snapshots).

> [!NOTE] Отличие от других VCS
> Git хранит данные как **набор снимков (snapshots)** мини-файловой системы, а не как список изменений (deltas) для каждого файла.
