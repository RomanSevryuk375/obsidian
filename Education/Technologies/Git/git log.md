Основной инструмент для инспекции истории [[git commit|коммитов]], позволяющий фильтровать, искать и визуализировать граф изменений в репозитории.

---

## Шпаргалка по флагам

```bash
# 1. Краткий обзор (ID и заголовок)
git log --oneline

# 2. Визуализация графа веток (Must have для DevOps)
# Показывает связи между ветками и слияния в консоли
git log --oneline --graph --all --decorate

# 3. Просмотр изменений в коде
# -p (patch) выводит diff для каждого коммита
git log -p -2          # Показать последние 2 коммита с кодом

# 4. Статистика изменений
# Показывает список измененных файлов и кол-во строк (+/-)
git log --stat

# 5. Поиск по автору или дате
git log --author="Ivan"
git log --since="2023-01-01" --until="yesterday"

# 6. Поиск по содержимому (Git Pickaxe)
# Найти коммиты, где в коде добавлялась или удалялась строка "API_KEY"
git log -S "API_KEY"

# 7. Поиск по сообщению коммита
# Помогает находить задачи, если используется [[Conventional Commits]]
git log --grep="feat(auth)"
```

## Кастомизация вывода
Для создания красивых отчетов можно использовать форматирование:
`git log --pretty=format:"%h - %an, %ar : %s"`
*   `%h`: короткий хеш.
*   `%an`: имя автора.
*   `%ar`: дата (relative).
*   `%s`: тема коммита.

## Связанные темы
*   [[git commit]] — создание записей, которые отображает log.
*   [[HEAD и указатели]] — понимание того, на какой коммит в логе мы смотрим.
*   [[git show]] — детальный просмотр одного конкретного коммита из списка.
*   [[git diff]] — сравнение состояний между коммитами из лога.
