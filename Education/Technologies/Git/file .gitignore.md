Текстовый файл в корне репозитория, определяющий намеренно неотслеживаемые файлы (логи, зависимости, секреты), которые [[Git]] должен игнорировать при выполнении [[git add]].

---

## Правила синтаксиса
*   `node_modules/` — игнорировать всю директорию.
*   `*.log` — игнорировать все файлы с расширением .log.
*   `.env` — игнорировать конкретный файл (обычно содержит секреты).
*   `dist/**/*.js` — игнорировать все `.js` файлы внутри `dist` и всех её подпапок.
*   `!important.log` — инверсия (НЕ игнорировать этот файл, даже если выше есть правило для всех `.log`).

## Шпаргалка

```bash
# 1. Создание файла (обычно делается сразу после [[git init]])
touch .gitignore

# 2. Если файл уже отслеживается (был закоммичен), .gitignore на него не подействует.
# Нужно "отвязать" его от индекса, сохранив физически на диске:
git rm --cached secrets.txt
git add .gitignore
git commit -m "chore: убрать секреты из отслеживания"

# 3. Проверить, почему файл игнорируется (какое правило сработало)
git check-ignore -v path/to/file.log

# 4. Глобальный ignore (для всей системы, например для .DS_Store или настроек IDE)
# Настраивается через
git config --global core.excludesfile ~/.gitignore_global
```

> [!TIP] DevOps Best Practice
> Всегда добавляйте `.env`, папки зависимостей (`vendor/`, `node_modules/`) и папки сборки (`dist/`, `build/`) в `.gitignore` до первого пуша в удаленный репозиторий, чтобы избежать утечки учетных данных и раздувания размера репозитория.

## Связанные темы
*   [[git add]] — команда, на которую напрямую влияет `.gitignore`.
*   [[git status]] — не показывает файлы, указанные в ignore.
*   [[git rm]] — используется для очистки репозитория от случайно попавших файлов.
*   [[Установка и настройка Git]] — для настройки глобальных правил игнорирования.
