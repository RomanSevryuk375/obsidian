# Git Map of Content

## 1. Фундамент и Базовая конфигурация
Основы архитектуры Git, первичная настройка окружения и команды жизненного цикла файлов.

- [[Что такое VCS и Git]] — понимание распределенной системы.
- [[Установка и настройка Git]] — конфиг `~/.gitconfig`, user.name, user.email.
- [[git init]] — инициализация локального репозитория.
- [[git clone]] — клонирование существующих репозиториев.
- [[git add]] — добавление изменений в Index (Staging Area).
- [[git commit]] — фиксация изменений и атомарность коммитов.
- [[git status]] — просмотр состояния рабочего дерева.
- [[git log]] — история изменений и форматирование вывода.
- [[file .gitignore]] — правила игнорирования файлов и директорий.
- [[git diff]] — просмотр разницы между состояниями.

## 2. Ветвление и Управление потоком
Изоляция разработки функционала, стратегии переключения контекста и объединения историй.

- [[git branch]] — создание, листинг и удаление веток.
- [[git checkout]] vs [[git switch]] — переключение между ветками (старый и новый синтаксис).
- [[HEAD и указатели]] — понимание того, как Git ссылается на текущее состояние.
- [[git merge]] — слияние веток (Fast-forward vs Merge Commit).
- [[Разрешение конфликтов слияния]] — алгоритмы и инструменты (kdiff3, p4merge).
- [[git tag]] — семантическое версионирование релизов.

## 3. Работа с удаленными репозиториями
Синхронизация локального кода с сервером (GitHub, GitLab, Bitbucket) и командная работа.

- [[git remote]] — управление удаленными источниками (origin, upstream).
- [[SSH ключи и аутентификация]] — безопасное подключение без паролей.
- [[git fetch]] — получение изменений без слияния.
- [[git pull]] — получение и автоматическое слияние (`fetch` + `merge`).
- [[git push]] — отправка локальных коммитов в удаленный репозиторий.
- [[Pull Requests / Merge Requests]] — процесс код-ревью (концепт платформ).

## 4. Отмена изменений и "Машина времени"
Техники безопасного и деструктивного отката изменений на разных этапах работы.

- [[git restore]] — восстановление файлов из индекса или коммита.
- [[git reset]] — (Soft, Mixed, Hard) перемещение HEAD и сброс индекса.
- [[git revert]] — безопасная отмена публичного коммита новым коммитом.
- [[git commit --amend]] — редактирование последнего коммита.
- [[git clean]] — удаление неотслеживаемых (untracked) файлов.
- [[git rm]] и [[git mv]] — правильное удаление и перемещение файлов в контексте Git.

## 5. Продвинутые техники и Инструментарий
Инструменты для поддержания чистоты истории, дебаггинга и сложной работы с зависимостями.

- [[git rebase]] — перебазирование веток для линейной истории.
- [[Interactive Rebase]] (`git rebase -i`) — сквош (squash), переименование и удаление коммитов.
- [[git cherry-pick]] — перенос конкретного коммита в другую ветку.
- [[git stash]] — временное сохранение "грязной" работы без коммита.
- [[git reflog]] — журнал ссылок (спасательный круг для потерянных коммитов).
- [[git bisect]] — бинарный поиск для нахождения коммита, сломавшего билд.
- [[git blame]] — построчный анализ авторства кода.
- [[git submodule]] — вложенные репозитории (плюсы и минусы).
- [[git worktree]] — одновременная работа с несколькими ветками в разных директориях.
- [[Git LFS]] — работа с большими бинарными файлами.
- [[Git Hooks]] — автоматизация (pre-commit, pre-push скрипты).

## 6. Методологии и Best Practices
Стандарты индустрии для организации командной разработки и CI/CD процессов.

- [[Git Flow]] — классическая модель ветвления для релизных циклов.
- [[GitHub Flow]] — упрощенная модель для CD.
- [[Trunk Based Development]] — современный подход для быстрой доставки.
- [[Conventional Commits]] — стандартизация сообщений коммитов для автоматизации changelog.
