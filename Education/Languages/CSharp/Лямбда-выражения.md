### Суть (The "What")
**Лямбда-выражение** — это анонимный метод, который используется для создания [[Делегаты|делегатов]] или деревьев выражений. Это краткая форма записи анонимных функций, позволяющая передавать блок кода как данные.

Лямбды решают задачу **снижения синтаксического шума**. Они избавляют от необходимости объявлять именованные методы для простых операций, что критично при работе с [[LINQ]], событиями и анонимными обратными вызовами (callbacks).

### Как это работает (The "How" / Nutshell style)

#### 1. Компиляция и генерация кода
На низком уровне лямбда-выражение не существует в рантайме как отдельная сущность. Компилятор [[MOC#/MOC|C#]] выполняет одну из двух стратегий трансформации:
*   **Статический метод:** Если лямбда не захватывает внешние переменные (замыкания), компилятор генерирует скрытый статический метод в текущем классе. Это максимально эффективно.
*   **Замыкание (Closure):** Если лямбда использует локальные переменные из внешнего метода, компилятор создает специальный приватный класс (обычно называемый `DisplayClass`). Внешние переменные становятся полями этого класса, а сама лямбда — методом этого класса.

#### 2. Захват переменных (Variable Capture)
Лямбды захватывают ==не *значение*== переменной на момент создания, а саму *переменную* (==ссылку на неё==). 
> [!IMPORTANT] Нюанс: Время жизни (Lifetime)
> Если лямбда захватывает локальную переменную, эта переменная переносится из стека в кучу (внутри `DisplayClass`). Это гарантирует, что переменная будет доступна, даже когда внешний метод завершит работу, но создает нагрузку на сборщик мусора ([[GC]]).

#### 3. Естественные типы и параметры ([[MOC#/MOC|C#]] 10+)
Начиная с C# 10, компилятор может пытаться вывести "естественный" тип лямбды (`Func` или `Action`) автоматически, что позволяет использовать `var`. В C# 12 добавлены **параметры по умолчанию** для лямбд.

#### 4. Expression Lambdas vs Statement Lambdas
*   **Выражения (Expression):** `(x) => x * x` — могут быть преобразованы в `Expression<TDelegate>`, что позволяет разбирать их код в рантайме (используется в Entity Framework для генерации SQL).
*   **Инструкции (Statement):** `(x) => { return x * x; }` — всегда компилируются только в делегаты.

### Пример кода (C# 12)

```csharp
using System;
using System.Collections.Generic;
using System.Linq;

public class LambdaDemo
{
    public static void Main()
    {
        // 1. Лямбда с параметром по умолчанию (C# 12)
        var greet = (string name = "Guest") => $"Hello, {name}!";
        Console.WriteLine(greet()); // Hello, Guest!

        // 2. Замыкание (Closure)
        int factor = 10;
        Func<int, int> multiplier = n => n * factor;
        
        factor = 20; // Изменение переменной влияет на результат лямбды
        Console.WriteLine(multiplier(5)); // Выведет 100, а не 50

        // 3. Статическая лямбда (C# 9+)
        // Гарантирует отсутствие захвата переменных, предотвращая аллокации
        var square = static (int x) => x * x;

        // 4. Использование в LINQ
        List<int> numbers = [1, 2, 3, 4, 5];
        var evenNumbers = numbers.Where(n => n % 2 == 0).ToList();
    }
}
```

### Ошибки и Best Practices

> [!DANGER] Anti-pattern: Захват итератора (до C# 5.0)
> В старых версиях C# захват переменной цикла в `for` приводил к тому, что все лямбды использовали последнее значение итератора. В современном C# (начиная с 5.0) внутри `foreach` создается "свежая" переменная на каждой итерации, но в классическом `for(int i...)` проблема все еще требует внимания.

> [!WARNING] Ошибка: Чрезмерная сложность
> Если лямбда содержит более 3-5 строк кода или сложную логику ветвления, её следует вынести в именованный метод. Лямбды предназначены для краткости; "простыни" кода внутри лямбд нечитаемы и трудноотлаживаемы.

**Best Practices:**
1.  **Static Lambdas:** Если лямбда не использует внешние данные, всегда помечайте её как `static` (C# 9+). Это предотвращает случайный захват `this` или локальных переменных и снижает нагрузку на кучу.
2.  **Explicit Types:** Если компилятор не может вывести типы параметров, указывайте их явно: `(int x, int y) => ...`.
3.  **Discard (_):** Если параметр лямбды не используется, используйте символ подчеркивания: `(_, args) => Handle()`.
4.  **Async Lambdas:** Будьте осторожны с `async void` лямбдами (например, в событиях). Старайтесь использовать `async Task` лямбды там, где это поддерживает вызывающий код.

> [!TIP] Нюанс памяти
> Создание лямбды — это создание объекта делегата. В "горячих" циклах (миллионы итераций) создание лямбд может привести к заметным паузам GC. В таких случаях лучше кэшировать делегат в статическом поле или использовать обычный метод.
