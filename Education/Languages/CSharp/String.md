### Суть (The "What")
`string` (синоним `System.String`) — это [[Reference Types#/Reference Types|ссылочный тип]] данных, представляющий собой неизменяемую (immutable) последовательность символов Юникода (UTF-16). 

Тип решает задачу безопасной и удобной работы с текстовой информацией. В отличие от массивов символов в неуправляемых языках, строки в [[MOC#/MOC|C#]] защищены от переполнения буфера, всегда знают свою длину и поддерживают богатый набор методов для манипуляции текстом.

---

### Как это работает (The "How" / Nutshell style)

#### 1. Неизменяемость (Immutability)
Это ключевая особенность: после создания объекта строки в [[Куча|куче]] его содержимое не может быть изменено. 
*   Любая операция, которая "изменяет" строку (например, `Replace` или `ToUpper`), на самом деле создает в куче **новый объект** строки и возвращает ссылку на него.
*   Это обеспечивает потокобезопасность: несколько потоков могут читать одну и ту же строку, не опасаясь, что кто-то её изменит.

#### 2. Интернирование (String Interning)
[[CLR]] оптимизирует использование памяти, сохраняя только один экземпляр каждой уникальной строковой константы в специальной таблице — **пуле интернирования (Intern Pool)**.
*   Если в коде объявлены две одинаковые литеральные строки (например, `string s1 = "hello"; string s2 = "hello";`), они будут указывать на один и тот же адрес в памяти.
*   Динамически созданные строки (например, через `ReadLine()`) не интернируются автоматически, но это можно сделать принудительно через `string.Intern()`.

#### 3. Устройство в памяти
Строка в .NET — это специальный объект. Помимо стандартного заголовка (8 байт на Sync Block и 8 байт на Type Handle), она содержит:
*   Длину строки (4 байта).
*   Массив символов (по 2 байта на каждый `char`).
*   Нулевой терминатор (`\0`) для совместимости с нативным кодом (C++).

> [!INFO] Нюанс: Сравнение
> Оператор `==` для строк переопределен. Несмотря на то что это ссылочный тип, `==` сравнивает не адреса, а **значения** (содержимое строк). Для сравнения ссылок используйте `object.ReferenceEquals()`.

#### 4. Память и производительность
Из-за неизменяемости конкатенация строк в цикле (через `+`) создает огромное количество временных объектов, которые нагружают сборщик мусора ([[GC]]). Для таких операций используется `StringBuilder` (изменяемый буфер) или современные типы `Span<char>` / `ReadOnlySpan<char>`.

### Пример кода (C# 12)

```csharp
using System;
using System.Text;

public class StringDemo
{
    public static void Main()
    {
        // 1. Разные виды литералов
        string path = @"C:\Users\Admin";    // Verbatim (игнорирует \ )
        string json = """{"id": 1}""";      // Raw string literal (C# 11+)
        
        // 2. Интерполяция (современный синтаксис)
        string name = "Artyom";
        string greeting = $"Hello, {name}!"; 

        // 3. StringBuilder для тяжелых операций
        var sb = new StringBuilder();
        for (int i = 0; i < 100; i++)
        {
            sb.Append(i).Append(' ');
        }
        string result = sb.ToString();

        // 4. Работа с памятью без аллокаций (Span)
        ReadOnlySpan<char> span = "Text to slice".AsSpan();
        ReadOnlySpan<char> slice = span.Slice(0, 4); // "Text" - новый объект не создан
        
        Console.WriteLine(slice.ToString());
    }
}
```

### Ошибки и Best Practices

> [!DANGER] Anti-pattern: Конкатенация в циклах
> Никогда не пишите `s += "next"` внутри цикла. Это приводит к сложности O(N²) по времени и памяти. Используйте [[StringBuilder]] или `string.Join()`.

> [!WARNING] Ошибка: Проверка на пустую строку
> Избегайте проверок вида `if (s == "" || s == null)`. Вместо этого используйте стандартные методы:
> - `string.IsNullOrEmpty(s)` — если пробелы считаются валидным текстом.
> - `string.IsNullOrWhiteSpace(s)` — если строка из одних пробелов считается пустой.

**Best Practices:**
1.  **Сравнение:** При сравнении строк для поиска или сортировки всегда указывайте `StringComparison` (например, `OrdinalIgnoreCase`), чтобы избежать багов, связанных с региональными стандартами (Culture).
2.  **Интерполяция:** Используйте `$"..."`, так как компилятор C# 10+ оптимизирует этот процесс через `DefaultInterpolatedStringHandler`, что часто быстрее и экономнее по памяти, чем `string.Format`.
3.  **Константы:** Если строка не меняется, помечайте её как `const`. Такие строки интернируются на этапе компиляции.
4.  **Срезы:** Если нужно "отрезать" часть строки для временного анализа, используйте `ReadOnlySpan<char>`. Это позволяет избежать копирования подстроки в новую область кучи.

> [!TIP] Нюанс: Длина
> Свойство `Length` возвращает количество символов `char` (UTF-16 юнитов), а не количество визуальных символов (графем). Для сложных символов (например, эмодзи или диакритических знаков) длина строки может не совпадать с количеством видимых знаков.
