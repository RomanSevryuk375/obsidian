### Суть (The "What")
[[StringBuilder]] — это динамический изменяемый буфер для работы с последовательностями символов. В отличие от [[String|string]], который является неизменяемым (immutable) типом, `StringBuilder` позволяет модифицировать содержимое без создания новых объектов в куче при каждой операции.

Он решает проблему квадратичной сложности $O(n^2)$ при массовой конкатенации строк. Каждая операция `s += "..."` создает новую строку, копируя все предыдущие данные, в то время как `StringBuilder` работает с уже выделенной памятью.

---

### Как это работает (The "How" / Nutshell style)

#### 1. Структура чанков (Chunks)
Начиная с .NET Core / .NET 4.0+, `StringBuilder` реализован не как один расширяемый массив, а как **обратный односвязный список** блоков (чанков).

Каждый объект `StringBuilder` содержит:
*   `char[] m_ChunkChars`: Текущий массив символов.
*   `StringBuilder m_ChunkPrevious`: Ссылка на предыдущий узел (предыдущий чанк).
*   `int m_ChunkLength`: Количество символов в текущем чанке.
*   `int m_ChunkOffset`: Суммарное количество символов во всех предыдущих чанках.

#### 2. Механика расширения
Когда текущий массив `m_ChunkChars` заполняется:
1.  Создается новый экземпляр `StringBuilder`.
2.  Новый экземпляр копирует в себя текущее состояние (становится "головой" списка).
3.  Старый экземпляр становится "предыдущим" (`m_ChunkPrevious`).
4.  Выделяется новый массив символов. Размер нового массива обычно удваивается, но ограничен 8Кб (чтобы не попадать в [[Large Object Heap]] слишком быстро).

> [!INFO] Нюанс: Доступ по индексу
> Из-за списочной структуры доступ к символу по индексу `sb[i]` перестал быть операцией $O(1)$. Теперь рантайму приходится проходить по цепочке чанков от конца к началу, пока не будет найден блок, содержащий нужный индекс. В худшем случае это $O(n)$ по количеству чанков.

#### 3. Генерация итоговой строки (ToString)
При вызове `ToString()` происходит следующее:
1.  Вычисляется общая длина (`m_ChunkOffset + m_ChunkLength`).
2.  В куче выделяется одна строка нужного размера.
3.  Происходит копирование данных из всех чанков в эту строку. Копирование идет "с конца", начиная с текущего чанка и двигаясь по ссылкам `m_ChunkPrevious`.

#### 4. Оптимизация интерполяции (C# 10+)
В современном [[MOC|C#]] вызов `sb.Append($"Value: {x}")` использует **Interpolated String Handlers**. Вместо того чтобы сначала собрать строку интерполяцией и передать её в метод, компилятор вызывает ряд методов `AppendLiteral` и `AppendFormatted` напрямую в буфер [[StringBuilder]], исключая создание промежуточной строки.

### Пример кода (C# 12)

```csharp
using System.Text;

public class PerformanceDemo
{
    public string EfficientBuild(int count)
    {
        // Указываем начальную емкость, чтобы минимизировать количество чанков
        var sb = new StringBuilder(capacity: 1024);

        for (int i = 0; i < count; i++)
        {
            // Используется AppendInterpolatedStringHandler (C# 10+)
            // Промежуточная строка для $"{i}" не создается
            sb.Append($"Item number: {i}; "); 
        }

        // Очистка для переиспользования (сохраняет выделенные чанки в памяти)
        // sb.Clear(); 

        return sb.ToString(); // Единственная точка аллокации итоговой строки
    }
}
```

### Ошибки и Best Practices

> [!DANGER] Anti-pattern: StringBuilder для коротких операций
> Использование `StringBuilder` для объединения 2-3 строк (например, `sb.Append(a).Append(b);`) медленнее, чем обычная конкатенация или интерполяция. Накладные расходы на создание объекта в куче и управление чанками перевешивают выгоду.

> [!WARNING] Ошибка: Неверный выбор начальной емкости
> Если `capacity` по умолчанию (16 символов) слишком мала, а итоговая строка будет огромной, возникнет множество чанков. Это замедлит `ToString()` и обращение по индексам. Всегда указывайте примерный размер, если он известен.

**Best Practices:**
1.  **Reusability:** В высокопроизводительных сценариях используйте `ObjectPool<StringBuilder>`, чтобы не создавать новые объекты и не провоцировать работу [[GC]].
2.  **Append(char) вместо Append(string):** Если нужно добавить один символ (например, точку с запятой), используйте `sb.Append(';')`. Это быстрее, так как не создает строковый литерал.
3.  **AppendJoin:** Для работы с коллекциями используйте `AppendJoin` — он оптимизирован для минимизации проверок границ и эффективно работает с разделителями.
4.  **MaxCapacity:** Используйте этот параметр в конструкторе, если нужно гарантировать, что буфер не разрастется бесконечно (защита от `OutOfMemoryException`).

> [!TIP] Нюанс: Многопоточность
> `StringBuilder` не является потокобезопасным. Если несколько потоков одновременно вызывают `Append`, внутреннее состояние чанков и индексов будет повреждено. Используйте внешнюю синхронизацию (`lock`).
