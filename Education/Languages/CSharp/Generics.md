### Суть (The "What")
**Обобщения** — это механизм параметрического [[Полиморфизм|полиморфизма]], позволяющий определять [[Классы и объекты|классы]], [[Интерфейсы|интерфейсы]] и [[Методы|методы]], в которых тип данных указывается как параметр (шаблон). 

Generics решают три фундаментальные задачи:
1.  **Безопасность типов (Type Safety):** Ошибки несовместимости типов выявляются на этапе компиляции, а не во время выполнения.
2.  **Производительность:** Устраняется необходимость в операциях упаковки/распаковки ([[boxing]]/[[unboxing]]) для значимых типов и приведении типов (casting) для ссылочных.
3.  **Повторное использование кода:** Один и тот же алгоритм может работать с разными типами данных без дублирования исходного кода.

### Как это работает (The "How" / Nutshell style)

#### 1. [[JIT]]-специализация (Низкий уровень)
В отличие от шаблонов C++, которые компилируются отдельно для каждого использования, обобщения в .NET сохраняются в метаданных сборки в "открытом" виде (например, `List<T>`). Фактическая генерация машинного кода происходит в рантайме (JIT):

*   **Для значимых типов (int, struct):** JIT-компилятор генерирует уникальный машинный код для каждой конкретной комбинации типов. Например, `Stack<int>` и `Stack<double>` — это две физически разные реализации в памяти. Это обеспечивает максимальную скорость работы с "сырыми" данными.
*   **Для ссылочных типов (string, class):** JIT-компилятор генерирует **одну общую** реализацию кода для всех ссылочных типов, так как все ссылки имеют одинаковый размер (указатель в 4 или 8 байт). Это экономит место в кеше инструкций процессора.

#### 2. Открытые и закрытые типы
*   **Открытый тип (Open Type):** Тип с неразрешенными параметрами, например `Dictionary<TKey, TValue>`. Вы не можете создать экземпляр открытого типа.
*   **Закрытый тип (Closed Type):** Тип, где все параметры определены, например `Dictionary<int, string>`.

#### 3. Статические члены
Каждый закрытый тип имеет собственный набор статических полей. 
> [!IMPORTANT] Нюанс памяти
> Если в классе `Counter<T>` есть `static int Count`, то у `Counter<int>.Count` и `Counter<string>.Count` будут разные ячейки памяти.

#### 4. Ограничения (Constraints)
С помощью ключевого слова `where` можно ограничить набор типов, которые могут быть переданы в качестве параметров:
*   `where T : struct` — значимый тип.
*   `where T : class` — ссылочный тип.
*   `where T : new()` — тип должен иметь публичный конструктор без параметров.
*   `where T : BaseClass` — тип должен быть наследником конкретного класса.
*   `where T : IInterface` — тип должен реализовывать интерфейс.
*   `where T : notnull` — тип не должен допускать null (C# 8+).

### Пример кода ([[MOC|C#]] 12)

```csharp
using System;

// Обобщенный интерфейс с ковариантностью (out)
public interface IRepository<out T> 
{
    T GetById(int id);
}

// Обобщенный класс с ограничениями и первичным конструктором (C# 12)
public class DataStore<T>(int initialCapacity) where T : class, new()
{
    private T[] _items = new T[initialCapacity];
    private int _count = 0;

    // Обобщенный метод
    public void Add(T item)
    {
        if (_count < _items.Length)
            _items[_count++] = item;
    }

    public T CreateDefault() => new T(); // Возможно благодаря where T : new()
}

public class Program
{
    public static void Main()
    {
        // Использование обобщенного класса (закрытый тип)
        var userStore = new DataStore<User>(10);
        userStore.Add(new User("Admin"));

        // Использование обобщенного метода расширения (вывод типа компилятором)
        Print(42); 
        Print("Hello Generics");
    }

    static void Print<T>(T value) => Console.WriteLine(value);
}

public record User(string Name);
```

### Ошибки и Best Practices

> [!DANGER] Anti-pattern: Использование object вместо обобщений
> Использование [[ArrayList]] или методов, принимающих `object`, приводит к потере типобезопасности и огромным накладным расходам на boxing (упаковку) значимых типов. Всегда используйте `List<T>` и другие обобщенные коллекции.

> [!WARNING] Ошибка: Чрезмерное количество ограничений
> Если для работы обобщенного класса требуется слишком много ограничений (`where T : I1, I2, I3, Base, new()`), это часто сигнализирует о нарушении принципа единственной ответственности или о том, что тип слишком жестко связан с иерархией классов.

**Best Practices:**
1.  **Имена параметров:** Используйте описательные имена с префиксом T (например, `TRequest`, `TResponse`), если параметров больше одного. Если параметр один — используйте просто `T`.
2.  **Интерфейсы коллекций:** В публичных API возвращайте или принимайте интерфейсы (`IEnumerable<T>`, `IReadOnlyList<T>`), а не конкретные реализации, чтобы сохранить гибкость.
3.  **Default(T):** Для получения значения по умолчанию используйте оператор `default(T)`. Для ссылочных типов это `null`, для значимых — "зануленная" структура.
4.  **Generic Math (C# 11+):** При работе с числами в обобщениях используйте интерфейсы типа `INumber<T>`, чтобы выполнять арифметические операции напрямую.

> [!TIP] Нюанс производительности
> В высоконагруженных участках кода использование обобщенных структур (struct) часто быстрее, чем классов, так как JIT генерирует для них специализированный код, который может быть инлайнен (встроен) агрессивнее.
