### Суть (The "What")
**Делегат** — это объектно-ориентированный, типобезопасный указатель на функцию. Он представляет собой [[Reference Types|ссылочный тип]], который определяет сигнатуру метода (тип возвращаемого значения и параметры) и может хранить ссылки на методы, соответствующие этой сигнатуре.

Делегаты решают задачу **косвенного вызова (indirection)**: они позволяют передавать методы как аргументы, хранить их в переменных и реализовывать механизмы обратного вызова (callbacks), отделяя вызывающий код от конкретной реализации метода.

### Как это работает (The "How" / Nutshell style)

#### 1. Внутренняя структура ([[CLR]] Internals)
Когда вы объявляете делегат в [[MOC#/MOC|C#]], компилятор генерирует полноценный класс, наследуемый от `System.MulticastDelegate` (который, в свою очередь, наследуется от `System.Delegate`). 

Экземпляр делегата содержит три ключевых поля:
*   **_target:** Ссылка на объект (экземпляр класса), если метод является экземплярным. Если метод статический, это поле равно `null`.
*   **_methodPtr:** Указатель на адрес метода в памяти (скомпилированный нативный код).
*   **_invocationList:** Массив ссылок на другие делегаты (используется для Multicast-делегатов).

#### 2. Multicast-делегаты (Групповые)
Делегаты поддерживают операторы `+` и `-`. При объединении нескольких методов создается новый экземпляр делегата, содержащий список вызовов (invocation list). Методы в списке вызываются **последовательно** в том порядке, в котором они были добавлены.

> [!INFO] Нюанс: Возвращаемое значение в Multicast
> Если делегат имеет возвращаемое значение (не `void`), то при вызове цепочки методов результатом станет значение, возвращенное **последним** методом в списке. Значения остальных методов будут проигнорированы.

#### 3. Встроенные типы (Func, Action, Predicate)
В современной разработке редко объявляют свои типы делегатов. Используются стандартные:
*   `Action<...>` — для методов, возвращающих `void`.
*   `Func<..., TResult>` — для методов, возвращающих значение.
*   `Predicate<T>` — для методов, возвращающих `bool` (проверка условия).

### Пример кода (C# 12)

```csharp
using System;

public class DelegateDemo
{
    // Определение пользовательского делегата (редко в современном C#)
    public delegate int Operation(int x, int y);

    public static void Main()
    {
        // 1. Использование встроенного Func
        Func<int, int, int> addFunc = (a, b) => a + b;
        
        // 2. Групповое преобразование (Method Group Conversion)
        // Метод Multiply соответствует сигнатуре Func<int, int, int>
        Func<int, int, int> multiplyFunc = Multiply;

        // 3. Multicast делегат
        Action<string> logger = message => Console.WriteLine($"Log: {message}");
        logger += message => Console.WriteLine($"Email: {message}");

        Execute(10, 5, addFunc);
        logger("Process completed"); // Вызовет обе лямбды
    }

    public static int Multiply(int x, int y) => x * y;

    public static void Execute(int x, int y, Func<int, int, int> op)
    {
        int result = op(x, y);
        Console.WriteLine($"Result: {result}");
    }
}
```

### Ошибки и Best Practices

> [!WARNING] Ошибка: NullReferenceException
> Если переменная делегата равна `null`, попытка её вызова приведет к исключению. Всегда используйте сокращенный вызов через оператор условия: `myDelegate?.Invoke(args)`.

> [!DANGER] Anti-pattern: Утечки памяти (Memory Leaks)
> Делегаты удерживают ссылку на объект (`_target`). Если долгоживущий объект (например, статический класс) подписан на делегат короткоживущего объекта, короткоживущий объект **не будет удален** сборщиком мусора, пока делегат не будет отписан или очищен.

**Best Practices:**
1.  **Action и Func:** По возможности используйте встроенные типы делегатов вместо объявления собственных. Это унифицирует код.
2.  **Анонимные методы vs Лямбды:** Используйте лямбда-выражения `(args) => { ... }`. Анонимный синтаксис `delegate(args) { ... }` считается устаревшим.
3.  **Использование в событиях:** Для реализации паттерна "Наблюдатель" используйте `event`. Событие — это инкапсулированная обертка над делегатом, запрещающая прямой доступ к списку вызовов извне.
4.  **Target-typed new:** В C# 9+ при инициализации делегатов можно использовать упрощенный синтаксис: `Operation op = Multiply;`.

> [!TIP] Нюанс: Замыкания (Closures)
> Когда лямбда-выражение использует локальную переменную из внешнего метода, компилятор создает скрытый класс-контейнер. Это переносит переменную из стека в кучу, чтобы она оставалась доступной даже после завершения работы внешнего метода. Будьте осторожны с производительностью в "горячих" циклах.
