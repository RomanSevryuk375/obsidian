### Суть (The "What")
**Анонимные типы** — это механизм [[MOC#/MOC|C#]], позволяющий инкапсулировать набор свойств только для чтения в один объект без необходимости предварительного явного определения [[Классы и объекты|класса]]. Тип объекта генерируется компилятором "на лету" и недоступен на уровне исходного кода.

Они решают задачу **временной группировки данных**, когда структура нужна только внутри одного метода (локально). Чаще всего это применяется в [[LINQ]]-запросах для проекции данных (`Select`), когда из сложной модели нужно извлечь лишь несколько полей.

---

### Как это работает (The "How" / Nutshell style)

#### 1. Компиляция в классы
Хотя в коде типа нет, [[CLR]] работает с анонимным типом как с обычным **ссылочным типом (class)**. 
*   Компилятор генерирует запечатанный (`sealed`) класс с именем вида `<>f__AnonymousType0`.
*   Все свойства анонимного типа помечаются как `readonly`. Начиная с C# 9, они ведут себя как `init`-свойства.
*   Если в одной сборке объявлены два анонимных типа с одинаковыми именами свойств, их типами и порядком следования, компилятор использует **один и тот же** сгенерированный класс.

#### 2. Структурное равенство
Компилятор автоматически переопределяет методы `Equals()` и `GetHashCode()`. 
*   Два экземпляра анонимного типа считаются равными, если все их свойства имеют одинаковые значения. 
*   Это отличает их от обычных классов, где сравнение по умолчанию идет по ссылке.

#### 3. Вывод имен (Projection Initializers)
Если при создании анонимного типа передается существующая переменная или свойство, имя свойства анонимного типа может быть выведено автоматически.

> [!INFO] Нюанс: Ограничение видимости
> Анонимные типы имеют область видимости `internal`. Вы не можете передать анонимный тип как параметр метода или вернуть его из метода (за исключением использования `object` или `dynamic`), так как вызывающий код не будет знать имени типа для обращения к его свойствам.

### Пример кода (C# 12)

```csharp
using System;
using System.Linq;
using System.Collections.Generic;

public class AnonymousTypesDemo
{
    public static void Main()
    {
        // 1. Ручное создание анонимного типа
        // Требуется использование var
        var user = new { Name = "Artyom", Age = 30 };
        
        // 2. Проекция данных в LINQ (основной сценарий)
        var source = new List<Employee>
        {
            new("Ivan", "Manager", 5000),
            new("Maria", "Dev", 7000)
        };

        var projections = source.Select(e => new 
        { 
            e.Name,            // Имя выведено автоматически
            IsHighPaid = e.Salary > 6000 
        });

        foreach (var item in projections)
        {
            Console.WriteLine($"{item.Name}: {item.IsHighPaid}");
        }

        // 3. Сравнение (структурное равенство)
        var p1 = new { X = 1, Y = 2 };
        var p2 = new { X = 1, Y = 2 };
        Console.WriteLine(p1 == p2); // False (оператор == не переопределен)
        Console.WriteLine(p1.Equals(p2)); // True (метод Equals переопределен)
    }
}

public record Employee(string Name, string Role, decimal Salary);
```

### Ошибки и Best Practices

> [!DANGER] Anti-pattern: Возврат анонимного типа через dynamic
> Возвращать анонимные типы из методов через `object` или `dynamic` — плохая практика. Это убивает типизацию и делает код хрупким. Если данные должны покинуть пределы метода, используйте `ValueTuple` или `record`.

> [!WARNING] Ошибка: Изменение свойств
> Свойства анонимного типа доступны только для чтения. Код `user.Age = 31;` не скомпилируется. Для создания измененной копии анонимного типа в C# нет встроенного механизма `with` (как у рекордов), поэтому придется создавать новый объект вручную.

**Best Practices:**
1.  **Только для локальных нужд:** Используйте анонимные типы исключительно внутри одного метода (обычно для LINQ).
2.  **Анонимные типы vs Кортежи:** Если вам нужно вернуть результат из метода, используйте кортеж `(string Name, int Age)`. Если вам нужна поддержка в LINQ-провайдерах (например, Entity Framework), анонимные типы предпочтительнее.
3.  **Имена свойств:** Всегда давайте понятные имена свойствам при проекции, если они не выводятся автоматически, так как это влияет на читаемость всего последующего кода.

> [!TIP] Нюанс: Типизация в LINQ
> Большинство LINQ-провайдеров (EF Core) анализируют деревья выражений (`Expression Tree`). Анонимные типы в `Select` помогают провайдеру понять, какие именно колонки нужно запросить из базы данных (SQL Projection), что значительно экономит трафик и память.
