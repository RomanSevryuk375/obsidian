### Суть (The "What")
Преобразование типов — это процесс приведения значения одного типа к другому типу. В [[C#]] все преобразования делятся на неявные (implicit), выполняемые компилятором автоматически, и явные (еxplicit), требующие явного указания целевого типа в коде (приведение типов / casting).

Этот механизм обеспечивает **совместимость и интероперабельность данных**. Он позволяет безопасно расширять разрядность чисел при вычислениях или принудительно сужать их, беря на себя риски потери точности или переполнения.

---

### Как это работает (The "How" / Nutshell style)

#### 1. Неявные преобразования (Widening)
Происходят автоматически, когда компилятор гарантирует отсутствие потери данных (например, из `int` в `long` или из `float` в `double`).
*   **Низкий уровень:** [[JIT]]-компилятор генерирует простую [[IL]]-инструкцию преобразования (например, `conv.i8` для перевода в 64-битное целое). Расширение разрядности (знаковое или беззнаковое) выполняется аппаратно на процессоре и практически бесплатно с точки зрения производительности.

#### 2. Явные преобразования (Narrowing)
Требуются при риске потери данных или точности (например, из `double` в `int` или из `long` в `short`). Используется синтаксис `(тип)значение`.
*   **Логика:** При сужении целых чисел лишние старшие биты просто отсекаются. При преобразовании вещественного числа в целое дробная часть отбрасывается (округление к нулю).

#### 3. Контексты checked и unchecked
По умолчанию явные преобразования целых чисел в C# выполняются в режиме **`unchecked`** (без проверки переполнения) [1]. Если результат сужения не помещается в целевой тип, старшие биты усекаются, а число циклически "заворачивается" (например, `(byte)257` станет `1`).
*   **Режим `checked`:** Если обернуть операцию в блок `checked { ... }`, JIT-компилятор сгенерирует инструкции, проверяющие флаг переполнения процессора [1]. При переполнении будет выброшено исключение `OverflowException` [1].

#### 4. Парсинг строк (Parse и TryParse)
Преобразование из `string` в базовые типы не может быть выполнено простым приведением.
*   **`Parse()`**: Анализирует строку и возвращает значение, но выбрасывает исключение, если формат неверный [1].
*   **`TryParse()`**: Безопасный метод, возвращающий `bool` [1]. В современном .NET (начиная с .NET 7/8) `TryParse` глубоко оптимизирован: он принимает `ReadOnlySpan<char>`, что позволяет парсить подстроки без выделения памяти в куче (zero-allocation).

#### 5. Обобщенный парсинг (`IParsable<T>`)
Начиная с .NET 7 (C# 11+), базовые типы реализуют интерфейсы `IParsable<T>` и `ISpanParsable<T>`. Это позволяет писать обобщенные алгоритмы, способные парсить любой тип из строки.

> [!INFO] Нюанс: Класс System.Convert
> Класс `Convert` предоставляет методы для преобразования между всеми базовыми типами. В отличие от явного приведения `(int)doubleValue` (которое просто отсекает дробь), `Convert.ToInt32(doubleValue)` выполняет округление к ближайшему четному числу (банковское округление).

### Пример кода (C# 12)

```csharp
using System;

public class ConversionDemo
{
    public static void Main()
    {
        // 1. Неявное (безопасное) преобразование
        int smallInt = 12345;
        long bigLong = smallInt; // Автоматическое расширение

        // 2. Явное преобразование в unchecked-контексте (по умолчанию)
        int largeInt = 300;
        byte smallByte = (byte)largeInt; // Старшие биты отсечены. Результат: 44 (300 - 256)

        // 3. Контекст checked для контроля переполнения
        try
        {
            checked
            {
                byte overflowByte = (byte)largeInt; // Выбросит исключение
            }
        }
        catch (OverflowException)
        {
            Console.WriteLine("Обнаружено переполнение при приведении типов!");
        }

        // 4. Оптимизированный парсинг через ReadOnlySpan (C# 10+)
        ReadOnlySpan<char> rawData = "ID:98765-ACTIVE".AsSpan();
        ReadOnlySpan<char> idSpan = rawData.Slice(3, 5); // "98765" без аллокации в куче

        if (int.TryParse(idSpan, out int parsedId))
        {
            Console.WriteLine($"Успешно распарсен ID: {parsedId}");
        }

        // 5. Обобщенный парсинг (C# 11+)
        double val = ParseValue<double>("123,45");
        Console.WriteLine($"Распарсенное double-значение: {val}");
    }

    // Обобщенный метод парсинга благодаря IParsable<T>
    public static T ParseValue<T>(string input) where T : IParsable<T>
    {
        return T.Parse(input, null);
    }
}
```

### Ошибки и Best Practices

> [!DANGER] Anti-pattern: Скрытое усечение данных
> Опасно выполнять явные приведения типов над важными данными (например, идентификаторами или финансовыми показателями) без проверки `checked` или валидации диапазонов. Это приводит к тихим логическим ошибкам в расчетах.

> [!WARNING] Ошибка: Аллокации при парсинге подстрок
> Если вам нужно распарсить число из середины длинной строки, не используйте `string.Substring()`, так как это создаст временную строку в куче. Всегда приводите строку к `ReadOnlySpan<char>` и используйте перегрузки `TryParse`, принимающие `ReadOnlySpan`.

**Best Practices:**
1.  **TryParse по умолчанию:** При работе со строками, полученными от внешних систем или пользователя, всегда отдавайте приоритет `TryParse` перед `Parse`, чтобы избежать дорогостоящих исключений [1].
2.  **Контроль checked:** Используйте блоки `checked` в финансовых или математически критичных алгоритмах, где переполнение недопустимо [1].
3.  **Пользовательские операторы:** Если вы создаете свои типы, вы можете определить ключевые слова `implicit operator` или `explicit operator`. Делайте оператор явным (`explicit`), если при преобразовании возможна потеря данных или генерация исключений.
4.  **Округление:** Помните о разнице между `(int)myDouble` (простое усечение) и `Convert.ToInt32(myDouble)` (банковское округление). Выбирайте способ осознанно.

> [!TIP] Нюанс: Преобразования Enum
> Приведение `enum` к его базовому числовому типу (например, `int`) — это явное приведение `(int)MyEnum.Value`. Оно абсолютно бесплатно на уровне IL и не приводит к [[Boxing Unboxing|боксингу]], так как перечисления физически хранятся как числа.
