### Суть (The "What")
`ArrayList` — это динамический [[Kлacc Array|массив]], который может хранить объекты любого типа. Он является частью пространства имен `System.Collections` и представляет собой устаревшую (legacy) коллекцию, которая использовалась до появления обобщений ([[Generics]]) в .NET 2.0.

Главная проблема, которую он решал в свое время — отсутствие необходимости заранее знать размер массива. Однако в современном [[MOC|C#]] `ArrayList` практически полностью вытеснен типом [[List]]`<T>`.

---

### Как это работает (The "How" / Nutshell style)

#### 1. Отсутствие типизации и [[Boxing]]
`ArrayList` работает с типом `object`. Это означает, что в одну и ту же коллекцию можно одновременно добавить `int`, `string` и экземпляр `class`.
*   **Низкий уровень:** Когда вы добавляете значимый тип (например, `int`), происходит **упаковка (boxing)**. Значение копируется из стека в кучу, создается объект-обертка, и ссылка на него сохраняется в массиве. 
*   **Извлечение:** При чтении требуется **распаковка ([[unboxing]])** — явное приведение типа, что создает дополнительную нагрузку на CPU.

#### 2. Динамическое изменение размера
Внутри `ArrayList` хранит массив `object[]`. 
*   Когда количество элементов достигает текущей емкости (`Capacity`), коллекция создает новый массив, размер которого обычно **в два раза больше** предыдущего.
*   Все существующие элементы копируются в новый массив. Это операция со сложностью $O(n)$.

#### 3. Безопасность типов (Type Safety)
`ArrayList` не обеспечивает безопасность типов на этапе компиляции. Ошибки несоответствия типов (например, попытка сложить число и строку, извлеченную из коллекции) проявляются только во время выполнения в виде `InvalidCastException`.

> [!WARNING] Нюанс: Потокобезопасность
> `ArrayList` не является потокобезопасным для операций записи. Для многопоточного доступа требуется ручная синхронизация через [[lock]] или использование метода `ArrayList.Synchronized()`, который создает медленную обертку (враппер) вокруг коллекции.

### Пример кода (C# 12)

Хотя использование `ArrayList` не рекомендуется, ниже показан пример его работы в сравнении с современными коллекциями.

```csharp
using System.Collections;

public class LegacyCollectionDemo
{
    public static void Main()
    {
        // Создание ArrayList (Legacy style)
        ArrayList list = new ArrayList();

        // 1. Boxing: int превращается в объект в куче
        list.Add(42); 
        
        // 2. Добавление разных типов в одну кучу (плохая практика)
        list.Add("Hello World");
        list.Add(new object());

        // 3. Извлечение данных требует явного приведения типов
        try
        {
            // Риск InvalidCastException
            int value = (int)list[0]!; 
            Console.WriteLine($"Value: {value}");
        }
        catch (InvalidCastException ex)
        {
            Console.WriteLine(ex.Message);
        }

        // 4. Использование современных выражений коллекций (C# 12)
        // Компилятор создаст List<object> или object[], если тип не указан явно
        object[] modernAlternative = [10, "Text", 3.14];
    }
}
```

### Ошибки и Best Practices

> [!DANGER] Anti-pattern: Использование ArrayList в новом коде
> Никогда не используйте `ArrayList` в современных проектах. Использование `List<object>` (если действительно нужна разнородная коллекция) или `List<T>` (в 99% случаев) работает быстрее за счет отсутствия упаковки и обеспечивает проверку типов на этапе компиляции.

> [!IMPORTANT] Ошибка: Производительность GC
> Обильное использование `ArrayList` со значимыми типами (числа, структуры) создает огромное количество мелких объектов в 0-м поколении кучи из-за упаковки. Это провоцирует частые и неэффективные циклы сборки мусора ([[Garbage Collection]]).

**Best Practices:**
1.  **Замена на List\<T\>:** Всегда заменяйте `ArrayList` на `List<T>`. Это стандарт индустрии.
2.  **Для разнородных данных:** Если вам действительно нужно хранить разные типы, используйте `List<object>`. Это делает намерения разработчика более явными и предоставляет доступ к методам [[LINQ]].
3.  **Утиная типизация:** Помните, что `ArrayList` реализует [[IEnumerable]], поэтому его можно перебирать через `foreach`, но переменная цикла должна иметь тип `object` или требовать явного приведения.

> [!TIP] Нюанс: Разница с массивом
> В отличие от обычного массива `object[]`, `ArrayList` всегда занимает больше памяти из-за объектных накладных расходов самого экземпляра коллекции и стратегии избыточного резервирования емкости (Capacity).
