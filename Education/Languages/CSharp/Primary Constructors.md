### Суть (The "What")
**Первичные конструкторы** — это синтаксическая конструкция [[MOC|C#]] 12, позволяющая объявлять параметры конструктора непосредственно в заголовке [[Классы и объекты|класса]] или [[Структуры|структуры]]. 

Они решают проблему **избыточного шаблонного кода (boilerplate)**. До их появления разработчику приходилось вручную объявлять закрытые поля, принимать аргументы в конструкторе и присваивать их этим полям. Теперь параметры первичного конструктора доступны во всем теле класса.

---

### Как это работает (The "How" / Nutshell style)

#### 1. Область видимости и "захват"
Параметры первичного конструктора находятся в области видимости **всего класса**. 
*   Если параметр используется только для инициализации полей или свойств, компилятор не создает для него дополнительного места в объекте.
*   **Захват (Capture):** Если параметр используется внутри методов или свойств, компилятор автоматически генерирует скрытое (private) поле для хранения этого значения.

#### 2. Отличие от рекордов ([[Record's]])
Важно не путать первичные конструкторы классов с первичными конструкторами `record`:
*   В **рекордах** параметры автоматически превращаются в публичные свойства (`public property`).
*   В **классах и структурах** параметры ведут себя как обычные параметры: они не становятся свойствами автоматически и по умолчанию являются **изменяемыми (mutable)** внутри класса.

#### 3. Цепочка вызовов
Если в классе определен первичный конструктор, все остальные конструкторы этого класса **обязаны** вызывать его через ключевое слово `this(...)`. Это гарантирует, что параметры первичного конструктора всегда будут инициализированы.

#### 4. Взаимодействие с базовым классом
Вызов базового конструктора (`base`) осуществляется прямо в объявлении класса:
`public class Derived(int x) : Base(x) { ... }`

> [!INFO] Нюанс: Порядок инициализации
> Инициализаторы полей экземпляра выполняются первыми и могут использовать параметры первичного конструктора. Только после этого управление может быть передано в тела дополнительных конструкторов.

### Пример кода (C# 12)

```csharp
using System;

// Базовый класс
public class Person(string name)
{
    public string Name { get; } = name;
}

// Производный класс с первичным конструктором
// Параметр 'service' будет "захвачен" в скрытое поле
// Параметр 'initialAge' используется только при инициализации свойства
public class Employee(string name, int initialAge, ILogger service) : Person(name)
{
    // Инициализация свойства параметром
    public int Age { get; private set; } = initialAge;

    public void Work()
    {
        // Параметр 'service' доступен здесь напрямую (компилятор создал скрытое поле)
        service.Log($"{Name} is working.");
    }

    // Дополнительный конструктор ОБЯЗАН вызвать первичный
    public Employee(string name, ILogger service) : this(name, 18, service)
    {
        Console.WriteLine("Secondary constructor called.");
    }
}

public interface ILogger { void Log(string message); }
```

### Ошибки и Best Practices

> [!DANGER] Anti-pattern: Дублирование полей
> Не объявляйте вручную поле `private readonly ILogger _logger;` и не присваивайте ему значение `_logger = logger;` в теле класса, если вы используете первичный конструктор. Это приведет к созданию двух полей в памяти: одного вашего и одного скрытого "захваченного" компилятором.

> [!WARNING] Ошибка: Изменяемость параметров
> Параметры первичного конструктора в классах не являются `readonly`. Если вы случайно измените значение параметра внутри метода, это изменение отразится на всех последующих обращениях к этому параметру в классе.

**Best Practices:**
1.  **Dependency Injection:** Первичные конструкторы идеально подходят для внедрения зависимостей в ASP.NET Core контроллерах и сервисах, сокращая код на 20-30%.
2.  **Валидация:** Поскольку у первичного конструктора нет явного тела, выполняйте валидацию аргументов либо в инициализаторах свойств, либо в методах перед использованием, либо используйте дополнительные конструкторы для сложной логики.
3.  **Именование:** Используйте стандартные имена для параметров (camelCase), как для обычных аргументов методов, так как технически они ими и являются.
4.  **Structs:** Используйте первичные конструкторы в структурах для создания компактных типов данных, работающих в стеке.

> [!TIP] Нюанс памяти
> Если вы используете параметр первичного конструктора только для инициализации свойства `public int Id { get; } = id;`, компилятор оптимизирует код и не будет создавать лишнего скрытого поля для `id`, используя только поле свойства.
