### Суть (The "What")
`BlockingCollection<T>` — это высокоуровневая потокобезопасная коллекция-конвейер из пространства имен `System.Collections.Concurrent`, реализующая классический паттерн проектирования **Producer-Consumer (Производитель-Потребитель)**.

Она решает две ключевые задачи многопоточности:
1.  **Блокировка при чтении (Blocking):** если коллекция пуста, поток-потребитель засыпает (блокируется), не нагружая процессор, пока поток-производитель не добавит новый элемент.
2.  **Ограничение емкости (Bounding):** если коллекция заполнена до установленного предела, потоки-производители блокируются и ждут, пока потребители не освободят место, забирая элементы. Это предотвращает неконтролируемое разрастание очереди в оперативной памяти.

---

### Как это работает (The "How" / Nutshell style)

#### 1. Обертка над конкурентными коллекциями
`BlockingCollection<T>` не хранит данные напрямую, а является оберткой над интерфейсом `IProducerConsumerCollection<T>`:
*   **Поведение по умолчанию:** используется [[ConcurrentQueue]]`<T>` (логика FIFO — первым пришел, первым ушел).
*   **Кастомизация:** в конструктор можно передать другие коллекции. Например, передав [[ConcurrentStack]]`<T>`, вы получите блокирующий стек (LIFO), а передав [[ConcurrentBag]]`<T>` — неупорядоченный блокирующий буфер.

#### 2. Механизм блокировки
Под капотом `BlockingCollection<T>` использует сигнальные примитивы (такие как [[Semaphore и SemaphoreSlim|SemaphoreSlim]]), работающие в гибридном режиме. При блокировке потоки не выполняют холостые циклы опроса (busy-waiting/spinning), а засыпают на уровне ОС, освобождая ядра процессора для других задач.

#### 3. Жизненный цикл конвейера
*   **`Add` / `TryAdd`**: добавление элементов производителем.
*   **`Take` / `TryTake`**: извлечение элементов потребителем.
*   **`CompleteAdding()`**: метод, сообщающий коллекции, что производители больше не будут добавлять данные. Это критически важно для завершения работы потребителей.
*   **`GetConsumingEnumerable()`**: возвращает блокирующий итератор. Он извлекает (удаляет) элементы один за другим. Когда вызывается `CompleteAdding()`, этот итератор завершает цикл `foreach` без генерации исключений.

> [!DANGER] Разница между GetConsumingEnumerable() и обычным итератором
> Использование обычного `foreach (var item in blockingCollection)` не извлекает элементы из очереди, а просто берет моментальный снимок (snapshot) текущего состояния коллекции без блокировки. Чтобы реализовать паттерн потребителя, всегда используйте `foreach (var item in blockingCollection.GetConsumingEnumerable())`.

### Пример кода ([[C#]] 12)

```csharp
using System;
using System.Collections.Concurrent;
using System.Threading.Tasks;

public class PipelineDemo
{
    public static async Task Main()
    {
        // Создаем конвейер с ограничением емкости в 5 элементов
        using BlockingCollection<int> pipeline = new(boundedCapacity: 5);

        // 1. Поток-потребитель
        Task consumer = Task.Run(() =>
        {
            Console.WriteLine("Потребитель: Запущен.");
            
            // GetConsumingEnumerable блокирует поток при отсутствии элементов
            // и автоматически завершит цикл после вызова CompleteAdding()
            foreach (var item in pipeline.GetConsumingEnumerable())
            {
                Console.WriteLine($"[Потребитель] Извлечено: {item}");
                Thread.Sleep(200); // Имитация обработки данных
            }
            
            Console.WriteLine("Потребитель: Все задачи обработаны. Завершение.");
        });

        // 2. Поток-производитель
        Task producer = Task.Run(() =>
        {
            Console.WriteLine("Производитель: Запущен.");
            for (int i = 1; i <= 15; i++)
            {
                Console.WriteLine($"[Производитель] Пытаюсь добавить: {i}");
                pipeline.Add(i); // Заблокирует поток, если буфер (5 элементов) заполнен
                Console.WriteLine($"[Производитель] Успешно добавлено: {i}");
            }
            
            // Сигнализируем о завершении наполнения
            pipeline.CompleteAdding();
            Console.WriteLine("Производитель: Все данные отправлены. Завершение.");
        });

        await Task.WhenAll(producer, consumer);
    }
}
```

### Ошибки и Best Practices

> [!DANGER] Ошибка: Забытый вызов CompleteAdding
> Если производитель закончит работу, но не вызовет `CompleteAdding()`, поток потребителя, выполняющий `GetConsumingEnumerable()`, зависнет в блокировке навечно, ожидая новых данных. Это приведет к утечке потоков и зависанию приложения.

> [!WARNING] Ошибка: Попытка добавления после CompleteAdding
> Любой вызов `Add()` после того, как был выполнен метод `CompleteAdding()`, приведет к генерации исключения `InvalidOperationException`. Если вы не уверены, закрыт ли конвейер, используйте безопасный метод `TryAdd()`.

**Best Practices:**
1.  **Используйте using:** `BlockingCollection<T>` реализует интерфейс `IDisposable`, так как внутри неё создаются нативные дескрипторы ядерных объектов синхронизации. Всегда очищайте коллекцию после завершения работы конвейера.
2.  **Задавайте boundedCapacity:** Всегда указывайте максимальную емкость буфера в конструкторе. Без ограничения емкости производители могут забить всю доступную оперативную память, генерируя объекты быстрее, чем потребители успевают их обрабатывать.
3.  **[[CancellationToken]]:** Почти все методы (`Add`, `Take`, `GetConsumingEnumerable`) принимают `CancellationToken`. Всегда передавайте токен отмены, чтобы иметь возможность экстренно завершить зависший конвейер (например, при выключении сервера).
4.  **Несколько потребителей:** `BlockingCollection` прекрасно работает со схемой "один производитель — много потребителей". Потоки-потребители будут безопасно конкурировать за элементы, и каждый элемент гарантированно достанется только одному из них.

> [!TIP] Метод Take vs TryTake
> Если вам не нужна жесткая блокировка потока, используйте `TryTake(out T item, TimeSpan timeout)`. Метод попытается забрать элемент, но если очередь пуста, он выйдет по таймауту, вернув `false`, что позволит потоку выполнить другую полезную работу.
