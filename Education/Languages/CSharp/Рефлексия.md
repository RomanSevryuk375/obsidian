### Суть (The "What")
**Рефлексия** — это механизм в .NET, позволяющий программе исследовать собственную структуру (метаданные), а также динамически управлять своим поведением во время выполнения (Runtime). С её помощью можно анализировать загруженные сборки, типы, методы, свойства, поля и атрибуты, а также создавать новые объекты и вызывать их члены "на лету".

Рефлексия решает задачу **динамического расширения и метапрограммирования**. Она является фундаментом для Dependency Injection ([[Dependency Injection|DI]]) контейнеров, сериализаторов (JSON/XML), ORM-систем (Entity Framework), тестовых фреймворков (xUnit/NUnit) и плагинных архитектур.

### Как это работает (The "How" / Nutshell style)

#### 1. [[Метаданные]] и Сборки
Сборки .NET содержат не только скомпилированный [[IL]]-код, но и богатые метаданные — таблицы, описывающие все типы, их члены, связи и атрибуты. Рефлексия — это интерфейс запросов к этим таблицам.

#### 2. Точка входа: System.Type
Каждому типу в .NET соответствует единственный объект класса `System.Type` (реальная реализация — скрытый внутренний класс `RuntimeType`).
*   Этот объект физически представляет структуру [[MethodTable]] типа в [[Куча|куче]] загрузчика (`Loader Heap`).
*   Получить `Type` можно тремя способами:
    1.  `typeof(MyClass)` — на этапе компиляции (статически).
    2.  `obj.GetType()` — у конкретного объекта в рантайме (динамически).
    3.  `Type.GetType("Namespace.MyClass")` — по строковому имени типа.

#### 3. Позднее связывание (Late Binding)
С помощью классов `Assembly`, `Module`, `MemberInfo`, `MethodInfo`, `PropertyInfo`, `FieldInfo` можно обойти компилятор:
*   **Создание объектов:** `Activator.CreateInstance(type)` находит конструктор в метаданных и выделяет память под объект.
*   **Вызов методов:** `MethodInfo.Invoke(obj, parameters)` выполняет метод. [[CLR]] осуществляет проверку типов параметров прямо перед вызовом.
*   **Доступ к приватным членам:** Рефлексия позволяет игнорировать модификаторы доступа (`private`, `protected`), предоставляя доступ к скрытым деталям реализации с помощью флагов `BindingFlags.NonPublic`.

#### 4. Производительность рефлексии
Рефлексия — очень медленный механизм по сравнению со статическим кодом:
*   **Парсинг строк:** Поиск членов по имени требует строкового сравнения в таблицах метаданных.
*   **Упаковка ([[Boxing Unboxing|Boxing]]):** `MethodInfo.Invoke` принимает массив `object[]` и возвращает `object`. Это приводит к упаковке всех передаваемых и возвращаемых значимых типов.
*   **Отсутствие оптимизаций [[JIT]]:** Динамические вызовы не могут быть встроены (inline) JIT-компилятором, а также требуют постоянных проверок безопасности (Security Demands).

> [!INFO] Оптимизация через делегаты
> Чтобы обойти ограничения производительности, медленный первый вызов рефлексии используют для создания быстрого типизированного делегата (например, через `Delegate.CreateDelegate` или `Expression Trees`). Последующие вызовы этого делегата будут работать практически со скоростью прямого вызова метода.

### Пример кода ([[MOC|C#]] 12)

```csharp
using System;
using System.Reflection;

// Кастомный атрибут для разметки методов
[AttributeUsage(AttributeTargets.Method)]
public class ExecuteMeAttribute(string description) : Attribute // Primary constructor (C# 12)
{
    public string Description { get; } = description;
}

public class Worker
{
    private readonly string _secretCode = "42-CLR-INTERNAL";

    [ExecuteMe("Этот метод запускается динамически через рефлексию")]
    public void Work() => Console.WriteLine("Worker is doing some job...");

    private void SecretWork() => Console.WriteLine($"Secret worker executed. Key: {_secretCode}");
}

public class Program
{
    public static void Main()
    {
        // 1. Получение типа в рантайме
        Type workerType = typeof(Worker);

        // 2. Динамическое создание объекта
        object workerInstance = Activator.CreateInstance(workerType)!;

        // 3. Сканирование методов и чтение атрибутов
        MethodInfo[] methods = workerType.GetMethods(BindingFlags.Public | BindingFlags.Instance);
        foreach (var method in methods)
        {
            var attribute = method.GetCustomAttribute<ExecuteMeAttribute>();
            if (attribute != null)
            {
                Console.WriteLine($"Найдена цель: {method.Name} ({attribute.Description})");
                // Динамический вызов
                method.Invoke(workerInstance, null);
            }
        }

        // 4. Обход инкапсуляции (чтение приватных членов)
        MethodInfo? privateMethod = workerType.GetMethod("SecretWork", BindingFlags.NonPublic | BindingFlags.Instance);
        privateMethod?.Invoke(workerInstance, null); // Вызов приватного метода
    }
}
```

### Ошибки и Best Practices

> [!DANGER] Ошибка: Нарушение инкапсуляции в бизнес-коде
> Использование рефлексии для чтения или изменения `private` полей сторонних классов — хрупкое решение. Авторы библиотек могут изменить приватные поля при любом обновлении, так как они не входят в публичный контракт. Ваш код сломается в рантайме без предупреждений компилятора.

> [!WARNING] Ошибка: Рефлексия в "горячих" циклах
> Никогда не вызывайте `Type.GetMethod()` или `method.Invoke()` внутри циклов с тысячами итераций. Это парализует производительность приложения. Кэшируйте полученные объекты `MethodInfo` или компилируйте их в делегаты.

**Best Practices:**
1.  **Source Generators как альтернатива:** В современном .NET (начиная с .NET 6/7/8) для задач метапрограммирования (сериализация, DI) отдавайте приоритет **генераторам исходного кода (Source Generators)**. Они анализируют код на этапе компиляции и генерируют быстрый статический код, избавляя приложение от рантайм-рефлексии.
2.  **Кэширование метаданных:** Если без рефлексии не обойтись, всегда кэшируйте результаты поиска типов и членов (например, в статических словарях).
3.  **typeof vs GetType:** Используйте `typeof(T)` вместо `new T().GetType()`, так как `typeof` оценивается компилятором на этапе сборки и работает мгновенно.
4.  **Безопасный вызов:** Помните, что исключения, возникшие внутри вызываемого через `method.Invoke()` метода, будут упакованы в `TargetInvocationException`. Реальное исключение будет находиться в свойстве `InnerException`.

> [!TIP] Нюанс: Получение сборки
> Вы можете загрузить внешнюю библиотеку (DLL) прямо в процессе работы программы с помощью `Assembly.LoadFrom("path.dll")`, найти в ней классы, реализующие определенный интерфейс, и запустить их. Это стандартный способ написания модульных плагинных систем.
