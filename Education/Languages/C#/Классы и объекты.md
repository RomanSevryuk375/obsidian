### Суть (The "What")
**Класс** — это ссылочный тип данных ([[Reference Types]]), представляющий собой чертеж или шаблон, который определяет структуру данных (поля) и поведение (методы) сущности. 
**Объект (экземпляр)** — это конкретный участок памяти в управляемой куче([[heap]]), созданный на основе этого чертежа.

Классы решают задачу **[[инкапсуляции]]**: объединения данных и логики в одну защищенную абстракцию, а также обеспечивают механизмы [[Наследование#/Наследование|наследования]] и [[Полиморфизм#/Полиморфизм|полиморфизма]].

### Как это работает (The "How" / Nutshell style)

#### 1. Жизненный цикл в памяти
Когда выполняется инструкция `new MyClass()`:
1.  **Вычисление размера:** [[CLR]] определяет объем памяти, необходимый для всех полей экземпляра (включая поля базовых классов) плюс накладные расходы.
2.  **Выделение в куче:** Выделяется непрерывный блок памяти.
3.  **Инициализация заголовка (Object Header):** 
    *   **Sync Block Index:** Используется для потокобезопасности (lock) и хеш-кода.
    *   **Type Object Pointer (Method Table Pointer):** Указатель на структуру данных в домене приложения, описывающую сам тип (методы, статические поля). Именно это позволяет CLR знать тип объекта во время выполнения.
4.  **Вызов конструктора:** Выполняется код инициализации. Сначала вызываются конструкторы базовых классов, затем текущего.
5.  **Возврат ссылки:** Адрес выделенного блока записывается в переменную в стеке.

#### 2. Члены класса
Помимо полей и методов, классы могут содержать:
*   **[[Свойства]] (Properties):** Синтаксический сахар над методами доступа (get/set).
*   **[[Конструкторы]]:** Логика инициализации. С [[MOC#/MOC|C#]] 12 активно используются **Primary Constructors** (первичные конструкторы).
*   **Финализаторы:** Код, вызываемый [[GC]] перед уничтожением (используется крайне редко).

#### 3. Ключевое слово this
`this` — это неявный указатель на текущий экземпляр. На уровне [[IL]]-кода (Intermediate Language) `this` передается как первый скрытый аргумент в каждый экземплярный метод.

> [!INFO] Нюанс: Накладные расходы (Overhead)
> Каждый объект в 64-битной системе имеет накладные расходы в 16 байт (Sync Block Index + Type Object Pointer). Даже пустой класс `class Empty {}` при создании экземпляра будет занимать в куче минимум 24 байта (16 на заголовок + выравнивание памяти).

### Пример кода (C# 12)

```csharp
using System;

// Использование первичного конструктора (C# 12 Primary Constructor)
public class User(string name, int age)
{
    // Поля (Fields) - обычно private
    private string _name = name;
    
    // Автосвойство с инициализацией только при создании (init)
    public int Age { get; init; } = age;

    // Свойство с логикой (Property)
    public string Name 
    { 
        get => _name;
        set => _name = string.IsNullOrWhiteSpace(value) ? "Unknown" : value;
    }

    // Метод (Behavior)
    public void PrintInfo() => Console.WriteLine($"User: {Name}, Age: {Age}");
}

public class Program
{
    public static void Main()
    {
        // Инстанцирование (создание объекта)
        User user1 = new("Artyom", 30);
        
        // Объектная инициализация
        User user2 = new("Maria", 25) 
        { 
            Name = "Mary" 
        };

        user1.PrintInfo();
        user2.PrintInfo();
    }
}
```

### Ошибки и Best Practices

> [!WARNING] Ошибка: Чрезмерное использование public полей
> Нарушение инкапсуляции. Поля всегда должны быть `private` или `protected`. Для доступа извне используйте свойства. Это позволяет добавить валидацию или логику логирования позже, не ломая контракт класса.

> [!DANGER] Anti-pattern: God Object (Божественный объект)
> Создание класса, который делает слишком много. Следуйте принципу **[[SRP]] (Single Responsibility Principle)**: один класс — одна ответственность. Если класс превышает 300-500 строк, это сигнал к рефакторингу.

**Best Practices:**
1.  **Read-only by default:** Если свойство не должно меняться после создания, используйте модификатор `init` вместо `set` и помечайте поля как `readonly`.
2.  **Primary Constructors:** Используйте первичные конструкторы в C# 12 для сокращения шаблонного кода (boilerplate) при передаче зависимостей.
3.  **Partial Classes:** Если класс генерируется автоматически (например, в инструментах БД) или становится слишком большим, используйте ключевое слово `partial`, чтобы разделить определение на несколько файлов.
4.  **Запечатывание (Sealing):** Помечайте классы как [[sealed]], если не планируете наследование. Это дает небольшое преимущество в производительности за счет виртуализации вызовов (devirtualization).

> [!TIP] Нюанс: Статические члены
> Статические поля и методы принадлежат самому классу (Type Object), а не его экземплярам. Они загружаются один раз при первом обращении к типу и живут до завершения работы домена приложения.
