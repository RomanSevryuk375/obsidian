### Суть (The "What")
**Кортеж** — это компактная структура данных, предназначенная для группировки нескольких разнородных значений в одну логическую единицу. В современном [[MOC#/MOC|C#]] (7.0+) под кортежами подразумеваются **ValueTuples** — значимые типы, которые обеспечивают высокую производительность и удобный синтаксис.

Кортежи решают задачу **возврата нескольких значений из метода** без использования громоздких `out`-параметров или создания специальных классов/структур для временных наборов данных.

### Как это работает (The "How" / Nutshell style)

#### 1. ValueTuple против System.Tuple
Важно различать старую и новую реализации:
*   **System.Tuple (устаревший):** Ссылочный тип (класс). Создает аллокации в куче, свойства доступны только для чтения (`Item1`, `Item2`).
*   **System.ValueTuple (современный):** Значимый тип (структура). Размещается в стеке, поля являются изменяемыми, поддерживает именование элементов.

#### 2. Хранение в памяти
Поскольку `ValueTuple` — это структура, при передаче кортежа происходит копирование его значений. Это эффективно для небольших наборов данных, так как исключает нагрузку на Garbage Collector ([[GC]]).

#### 3. Имена элементов и стирание типов (Type Erasure)
Имена элементов кортежа (например, `(int count, string sum)`) существуют только на этапе компиляции.
*   На уровне [[IL]]-кода элементы всегда называются `Item1`, `Item2` и т.д.
*   Компилятор использует атрибут `[TupleElementNames]` для сохранения информации об именах в метаданных, чтобы другие сборки могли их распознать.
*   **Следствие:** Два кортежа `(int x, int y)` и `(int a, int b)` являются одним и тем же типом для CLR.

#### 4. Структурное равенство
Кортежи поддерживают **структурное сравнение**. Два кортежа считаются равными, если их соответствующие элементы равны. Имена полей при сравнении игнорируются.

> [!INFO] Нюанс C# 12: Псевдонимы (Aliasing)
> Теперь можно использовать директиву `using` для создания псевдонимов кортежей, что упрощает их переиспользование: `using Point = (int X, int Y);`.

### Пример кода (C# 12)

```csharp
using System;

// C# 12 Alias для кортежа
using Result = (bool IsSuccess, string Message, int Code);

public class TupleProcessor
{
    public static void Main()
    {
        // 1. Создание и инициализация
        var data = (Id: 1, Title: "Note");
        
        // 2. Использование метода, возвращающего кортеж
        var (isSuccess, message, code) = ExecuteTask(); // Деконструкция

        // 3. Игнорирование части данных через discard (_)
        var (onlySuccess, _, _) = ExecuteTask();

        // 4. Сравнение кортежей (по значениям, а не по именам)
        (int a, int b) p1 = (1, 10);
        (int x, int y) p2 = (1, 10);
        Console.WriteLine(p1 == p2); // True

        // 5. Использование алиаса C# 12
        Result finalResult = (true, "Operation completed", 200);
        Console.WriteLine(finalResult.Message);
    }

    private static Result ExecuteTask()
    {
        // Логика...
        return (true, "Success", 200);
    }
}
```

### Ошибки и Best Practices

> [!DANGER] Anti-pattern: Кортежи в публичном API библиотек
> Не рекомендуется использовать кортежи в качестве возвращаемых типов для публичных методов больших библиотек или SDK. Изменение состава кортежа ломает бинарную совместимость. В таких случаях лучше использовать `record` или `class`.

> [!WARNING] Ошибка: Boxing (Упаковка)
> Если кортеж `ValueTuple` приводится к интерфейсу (например, `IComparable`) или к `object`, происходит упаковка в кучу. Это нивелирует преимущество значимого типа.

**Best Practices:**
1.  **Локальность:** Используйте кортежи только для промежуточных вычислений и внутри приватных методов.
2.  **Имена:** Всегда давайте осмысленные имена элементам. Избегайте обращений через `Item1`, `Item2`, так как это делает код магическим и хрупким.
3.  **Деконструкция:** Используйте синтаксис деконструкции `var (a, b) = ...` для повышения читаемости.
4.  **Короткие цепочки:** Если кортеж содержит более 3-4 элементов, это сигнал о необходимости создания полноценного типа (`record` или `struct`).

> [!TIP] Нюанс: Изменяемость
> Поля `ValueTuple` — это открытые поля (`public fields`), а не свойства. Это сделано для максимальной производительности, но противоречит общему правилу инкапсуляции. Помните об этом при передаче кортежей в многопоточной среде.
