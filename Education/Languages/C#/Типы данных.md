### Суть (The "What")
[[MOC#/MOC|C#]] является **сильно типизированным (strongly typed)** языком. Это означает, что каждая переменная и каждая константа имеют строго определенный тип, который описывает объем выделяемой памяти, диапазон допустимых значений и набор операций, которые можно выполнять над данными.

Система типов решает задачу **безопасности типов (type safety)**: предотвращает некорректные операции (например, деление строки на число) и обеспечивает предсказуемость выделения ресурсов во время выполнения.

### Как это работает (The "How" / Nutshell style)

В основе системы типов .NET лежит **[[CTS]] (Common Type System)**. Все типы иерархически наследуются от `System.Object`.

#### 1. Классификация по способу хранения
Ключевое различие между типами заключается в том, как [[CLR]] (Common Language Runtime) управляет их памятью:

*   **Значимые типы ([[Value Types]]):**
    *   Включают все числовые типы, `bool`, `char`, а также пользовательские `struct` и `enum`.
    *   Переменная содержит **непосредственно данные**.
    *   Обычно размещаются в **стеке** (если являются локальными переменными) или внутри объекта в куче (если являются полями класса).
    *   При копировании создается полная копия данных.
    *   Наследуются от `System.ValueType`.

*   **Ссылочные типы ([[Reference Types]]):**
    *   Включают `class`, `interface`, `delegate`, `record`, `string` и массивы.
    *   Переменная содержит **адрес (ссылку)** на объект в управляемой куче (Heap).
    *   При копировании копируется только ссылка, а не сам объект.
    *   Управляются сборщиком мусора ([[Garbage Collector]]).

#### 2. Маппинг встроенных типов
C# предоставляет ключевые слова, которые являются "синонимами" типов из пространства имен `System`. Например:
- `int` -> `System.Int32`
- `string` -> `System.String`
- `decimal` -> `System.Decimal`

> [!INFO] Нюанс: Примитивы
> С точки зрения C#, примитивными считаются типы, которые можно выразить литералами (напр. `10`, `true`). Важно помнить, что `decimal` не является примитивом в CLR (он реализован как структура), хотя C# позволяет использовать его как встроенный тип.

#### 3. Числовые типы и точность
- **Целочисленные:** `byte`, `int`, `long` и их беззнаковые аналоги. Начиная с .NET 5+, добавлены `nint` и `nuint` для работы с указателями нативной разрядности.
- **Числа с плавающей запятой:** 
    - `float` (32 bit) и `double` (64 bit) используют двоичную логику (IEEE 754), что приводит к погрешностям при представлении десятичных дробей.
    - `decimal` (128 bit) использует десятичную логику, обеспечивая высокую точность для финансовых вычислений.

### Пример кода (C# 12)

```csharp
using System;

// Демонстрация разницы в поведении типов
public class DataTypeDemo
{
    public static void Main()
    {
        // 1. Значимый тип (Value Type)
        int a = 10;
        int b = a; // Копирование значения
        b = 20;
        Console.WriteLine($"a: {a}, b: {b}"); // a остается 10

        // 2. Ссылочный тип (Reference Type)
        PointRef p1 = new(0, 0);
        PointRef p2 = p1; // Копирование ссылки
        p2.X = 100;
        Console.WriteLine($"p1.X: {p1.X}, p2.X: {p2.X}"); // Оба 100

        // 3. Использование nint (Native-sized integer) - C# 9+
        nint nativeInt = 42; 
        
        // 4. Литералы и типы C# 12
        var buffer = (ReadOnlySpan<byte>)[1, 2, 3, 4, 5]; // Collection expression
    }
}

public class PointRef(int x, int y) // Primary constructor (C# 12)
{
    public int X { get; set; } = x;
    public int Y { get; set; } = y;
}
```

### Ошибки и Best Practices

> [!WARNING] Ошибка: Использование double для денег
> Никогда не используйте `float` или `double` для финансовых расчетов. Накопленная ошибка округления в двоичной системе приведет к потере точности. Используйте только `decimal`.

> [!DANGER] Anti-pattern: Boxing/Unboxing
> Избегайте преобразования значимых типов в `object` и обратно (например, `object o = 5;`). Это приводит к **упаковке ([[boxing]])** — выделению памяти в куче, что снижает производительность при частых итерациях.

**Best Practices:**
1.  **Выбор типа по умолчанию:** Для целых чисел используйте `int` (оптимален для 32/64-битных систем), для дробных — `double`, если не нужна специфическая точность `decimal`.
2.  **Immutability:** Для ссылочных типов, представляющих собой простые данные, используйте `record`. Это делает поведение ссылочного типа похожим на значимый за счет сравнения по значениям и неизменяемости.
3.  **Nullable типы:** Используйте `T?` (например, `int?`), если значение может отсутствовать. Это предотвращает `NullReferenceException`.
4.  **Checked-контекст:** При выполнении критических арифметических операций со значимыми типами используйте блок `checked { ... }`, чтобы получить исключение при переполнении, вместо "тихого" сброса счетчика.

> [!TIP] Нюанс памяти
> Массив значимых типов (например, `int[]`) хранится в куче как единый блок, но сами элементы внутри массива расположены последовательно как "чистые" значения, что крайне эффективно для кэша процессора.
