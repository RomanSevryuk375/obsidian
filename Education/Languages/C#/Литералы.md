### Суть (The "What")
**Литералы** — это фиксированные значения, записанные непосредственно в исходном коде программы. Они представляют собой константы, тип которых определяется компилятором на основе их синтаксиса.

Литералы решают задачу инициализации переменных и передачи константных данных в методы без необходимости создания именованных констант. Это "фундаментальные кирпичи", из которых строятся выражения.

---

### Как это работает (The "How" / Nutshell style)

#### 1. Типизация по умолчанию
Компилятор [[MOC#/MOC|C#]] автоматически назначает тип литералу, исходя из его формата:
*   `100` — по умолчанию `int`. Если значение превышает диапазон `int`, компилятор пробует `uint`, `long`, `ulong`.
*   `1.5` — по умолчанию `double`.
*   `true` / `false` — `bool`.
*   `'A'` — `char`.
*   `"text"` — `string`.

#### 2. Суффиксы (Type Suffixes)
Для явного указания типа, отличного от стандартного, используются суффиксы:
*   **Целые числа:** `U` (uint), `L` (long), `UL` (ulong).
*   **Числа с плавающей точкой:** `F` (float), `D` (double), `M` (decimal). 

> [!IMPORTANT] Нюанс: Decimal
> Литералы типа `decimal` обязаны иметь суффикс `M` или `m`. Без него число `1.0` будет воспринято как `double`, и попытка присвоить его переменной `decimal` вызовет ошибку компиляции из-за потери точности.

#### 3. Форматы записи чисел
*   **Шестнадцатеричный:** префикс `0x` (напр. `0xFF`).
*   **Двоичный:** префикс `0b` (напр. `0b1010`).
*   **Разделитель разрядов:** символ подчеркивания `_` (напр. `1_000_000`) доступен для всех числовых литералов и не влияет на значение, улучшая читаемость.

#### 4. Строковые литералы (C# 11 и 12)
*   **Verbatim (`@`):** Игнорирует управляющие последовательности (например, `\n`). Удобно для путей: `@"C:\Temp"`.
*   **Raw String Literals (`"""`):** Позволяют записывать многострочный текст и кавычки без экранирования. Количество открывающих кавычек определяет формат.
*   **UTF-8 литералы (`u8`):** Создают массив байт (`ReadOnlySpan<byte>`) непосредственно из строки.

### Пример кода (C# 12)

```csharp
using System;

public class LiteralDemo
{
    public static void Main()
    {
        // 1. Числовые литералы с разделителями
        long population = 8_000_000_000L; 
        double price = 19.99D;
        decimal balance = 1500.50M;
        int hex = 0xAF;
        int binary = 0b1010_1011;

        // 2. UTF-8 строковый литерал (C# 11)
        // Тип: ReadOnlySpan<byte>
        var utf8Buffer = "Hello"u8;

        // 3. Raw String Literal (C# 11)
        // Сохраняет форматирование и кавычки "как есть"
        string xml = """
                     <root>
                        <child id="1">Content</child>
                     </root>
                     """;

        // 4. Символьные литералы
        char slash = '\\';
        char heart = '\u2764'; // Unicode

        // 5. Литерал по умолчанию (C# 7.1+)
        int defaultInt = default; // 0
        string? defaultStr = default; // null
    }
}
```

### Ошибки и Best Practices

> [!DANGER] Anti-pattern: Magic Numbers
> Избегайте использования числовых литералов (например, `3.14` или `86400`) непосредственно в бизнес-логике. Выносите их в именованные константы: `private const double Pi = 3.14;`. Это улучшает читаемость и упрощает рефакторинг.

> [!WARNING] Ошибка: Точность Float/Double
> Помните, что литералы `0.1f + 0.2f` не будут равны `0.3f` из-за особенностей двоичного представления дробей. Для финансовых вычислений всегда используйте суффикс `M` (decimal).

**Best Practices:**
1.  **Разделители разрядов:** Всегда используйте `_` для больших чисел (начиная с 10 000), чтобы снизить вероятность ошибки при чтении кода.
2.  **Raw Strings для JSON/SQL:** Используйте `"""` для внедренных SQL-запросов или JSON-строк. Это избавляет от нагромождения символов `\"` и `\n`.
3.  **Верхний/Нижний регистр:** Суффиксы можно писать в обоих регистрах (`10L` или `10l`), но предпочтительнее верхний регистр (особенно для `L`), так как строчная `l` похожа на цифру `1`.
4.  **u8 для производительности:** В сетевых протоколах или при работе с вебом используйте `u8`, чтобы избежать лишней перекодировки строки из UTF-16 (стандарт [[MOC#/MOC|C#]]) в UTF-8 в рантайме.

> [!TIP] Нюанс: null
> Литерал `null` не имеет типа, но может быть приведен к любому ссылочному типу или Nullable-типу. Начиная с C# 10, для `null` доступны проверки через pattern matching: `if (obj is null)`.
